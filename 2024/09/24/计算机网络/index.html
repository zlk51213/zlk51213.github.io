计算机网络概述

概念

计算机网络是指将地理位置不同的具有独立功能的多个计算机及其外部设备，通过通信线路连接起来，在网络操作系统，网络管理软件及网络通信协议的管理和协调下，实现资源共享和信息传递的计算机系统。

网络分类

按照网络覆盖范围分类

  1. 局域网（LAN）：覆盖范围在几百米到十几公里以内，用于连接家庭、办公室、学校等小范围内的计算机设备。
  2. 城域网（MAN）：覆盖范围在几公里到几十公里以内，用于连接城市或地区内的计算机设备。
  3. 广域网（WAN）：覆盖范围在几十公里到几千公里以上，用于连接不同城市、地区或国家的计算机设备。

按照网络拓扑结构分类

  1. 星型拓扑：所有设备都连接到一个中央节点，中央节点负责数据转发。
  2. 环型拓扑：设备通过环形线路连接，数据沿环行线路传输。
  3. 总线拓扑：所有设备都连接到一条公共线路，数据沿线路传输。
  4. 网状拓扑：设备通过多条线路连接，数据可以沿多条路径传输。

按照网络传输介质分类

  1. 有线网络：使用物理介质（如电缆、光纤）进行数据传输。
  2. 无线网络：使用电磁波进行数据传输。

按照网络协议分类

  1. TCP/IP网络：使用TCP/IP协议栈进行数据传输。
  2. OSI网络：使用OSI模型进行数据传输。

网络协议

网络协议是计算机网络中不同设备之间进行通信的规则和约定。它定义了数据传输的格式、顺序、错误检测和纠正等。常见的网络协议包括：

  1. IP协议：互联网协议，用于在网络中传输数据包。
  2. TCP协议：传输控制协议，用于在网络上可靠地传输数据。
  3. UDP协议：用户数据报协议，用于在网络上快速传输数据。
  4. HTTP协议：超文本传输协议，用于在互联网上传输超文本数据。
  5. FTP协议：文件传输协议，用于在网络上传输文件。
  6. SMTP协议：简单邮件传输协议，用于在网络上传输电子邮件。
  7. DNS协议：域名系统协议，用于将域名解析为IP地址。

网络协议三要素

  1. 语法：数据与控制信息的结构和格式
  2. 语义：需要发出何种控制信息，完成何种动作以及做出何种响应
  3. 时序：事件发生顺序的详细说明

网络体系结构

OSI七层模型

OSI（Open System Interconnection，开放系统互连）模型是一个七层模型，用于描述计算机网络中不同层次的功能和交互。OSI模型将网络通信分为七个层次，从低到高依次为：

  1. 物理层（Physical Layer）：负责物理传输介质上的比特流传输，包括电气特性、信号编码、传输速率等。
  2. 数据链路层（Data Link Layer）：负责在两个相邻节点之间建立、维护和断开数据链路，包括错误检测和纠正、帧的封装和拆封等。
  3. 网络层（Network Layer）：负责数据包的传输和路由选择，包括IP地址、路由协议、网络拥塞控制等。
  4. 传输层（Transport Layer）：负责端到端的可靠数据传输，包括TCP、UDP等协议。
  5. 会话层（Session Layer）：负责建立、管理和终止会话，包括同步、检查点等。
  6. 表示层（Presentation Layer）：负责数据格式的转换和加密解密，包括字符编码、数据压缩等。
  7. 应用层（Application Layer）：负责提供各种网络服务，如HTTP、FTP、SMTP等。

TCP/IP四层模型

TCP/IP（Transmission Control Protocol/Internet Protocol，传输控制协议/互联网协议）模型是一个四层模型，用于描述计算机网络中不同层次的功能和交互。TCP/IP模型将网络通信分为四个层次，从低到高依次为：

  1. 链路层（Link Layer）：负责在两个相邻节点之间建立、维护和断开数据链路，包括错误检测和纠正、帧的封装和拆封等。
  2. 网络层（Network Layer）：负责数据包的传输和路由选择，包括IP地址、路由协议、网络拥塞控制等。
  3. 传输层（Transport Layer）：负责端到端的可靠数据传输，包括TCP、UDP等协议。
  4. 应用层（Application Layer）：负责提供各种网络服务，如HTTP、FTP、SMTP等。

五层模型

五层模型是一种简化的网络体系结构模型，将OSI七层模型和TCP/IP四层模型进行了合并和简化。五层模型将网络通信分为五个层次，从低到高依次为：

  1. 物理层（Physical Layer）：负责物理传输介质上的比特流传输，包括电气特性、信号编码、传输速率等。
  2. 数据链路层（Data Link Layer）：负责在两个相邻节点之间建立、维护和断开数据链路，包括错误检测和纠正、帧的封装和拆封等。
  3. 网络层（Network Layer）：负责数据包的传输和路由选择，包括IP地址、路由协议、网络拥塞控制等。
  4. 传输层（Transport Layer）：负责端到端的可靠数据传输，包括TCP、UDP等协议。
  5. 应用层（Application Layer）：负责提供各种网络服务，如HTTP、FTP、SMTP等。

计算机网络的性能指标

  1. 速率：数据传输速率，单位为比特每秒（bps）。
  2. 带宽：网络中某条链路的最高数据传输速率，单位为比特每秒（bps）。
  3. 吞吐量：网络中某条链路在单位时间内实际传输的数据量，单位为比特每秒（bps）。
  4. 时延：数据从发送端到接收端所需的时间，包括发送时延、传播时延、处理时延和排队时延。
  5. 时延带宽积：时延带宽积等于带宽乘以时延，表示网络中某条链路的缓冲区大小。
  6. 往返时间（RTT）：数据从发送端发送到接收端，再从接收端返回到发送端所需的时间。
  7. 利用率：网络中某条链路被实际数据传输占用的比例。

网络设备

网络设备是计算机网络中用于实现数据传输和路由的设备。常见的网络设备包括：

  1. 路由器（Router）：用于连接不同的网络，实现数据包的路由和转发。
  2. 交换机（Switch）：用于连接计算机和其他网络设备，实现数据包的交换。
  3. 集线器（Hub）：用于连接计算机和其他网络设备，实现数据包的广播。
  4. 网桥（Bridge）：用于连接两个或多个网络，实现数据包的转发。
  5. 网关（Gateway）：用于连接不同的网络，实现不同网络协议之间的转换。
  6. 防火墙（Firewall）：用于保护网络免受外部攻击，实现数据包的过滤和转发。
  7. 负载均衡器（Load Balancer）：用于将网络流量分配到多个服务器，提高网络性能和可靠性。

网络安全

网络安全是计算机网络中保护数据传输和系统安全的重要方面。常见的网络安全措施包括：

  1. 加密：使用加密算法对数据进行加密，确保数据在传输过程中的安全性。
  2. 认证：使用身份验证机制，确保只有授权用户才能访问网络资源。
  3. 访问控制：使用访问控制列表（ACL）限制对网络资源的访问。
  4. 防火墙：使用防火墙阻止未经授权的访问和数据传输。
  5. 入侵检测系统（IDS）：监控网络流量，检测和报告潜在的入侵行为。
  6. 入侵防御系统（IPS）：阻止和防御已知的入侵行为。
  7. 安全审计：定期审计网络和系统安全，发现和修复安全漏洞。

网络性能优化

网络性能优化是提高网络传输速度和可靠性的重要方面。常见的网络性能优化措施包括：

  1. 优化网络拓扑：通过优化网络拓扑结构，减少网络延迟和提高网络带宽。
  2. 优化路由算法：通过优化路由算法，选择最优路径进行数据传输。
  3. 优化数据传输协议：通过优化数据传输协议，提高数据传输效率和可靠性。
  4. 优化网络设备配置：通过优化网络设备配置，提高网络设备的性能和可靠性。
  5. 优化网络带宽：通过优化网络带宽，提高网络传输速度和可靠性。
  6. 优化网络流量管理：通过优化网络流量管理，减少网络拥塞和提高网络性能。
  7. 优化网络监控：通过优化网络监控，及时发现和解决网络性能问题。

网络故障排除

网络故障排除是解决网络问题的重要方面。常见的网络故障排除方法包括：

  1. 检查网络连接：检查网络连接是否正常，包括物理连接和网络配置。
  2. 检查网络设备：检查网络设备是否正常工作，包括路由器、交换机和防火墙等。
  3. 检查网络协议：检查网络协议是否正常工作，包括IP协议、TCP协议和UDP协议等。
  4. 检查网络流量：检查网络流量是否正常，包括网络带宽和流量分布等。
  5. 检查网络日志：检查网络日志，发现和解决网络问题。
  6. 使用网络诊断工具：使用网络诊断工具，如ping、traceroute和netstat等，诊断网络问题。
  7. 检查网络配置：检查网络配置是否正确，包括IP地址、子网掩码和网关等。

网络管理

网络管理是确保网络正常运行和维护网络性能的重要方面。常见的网络管理任务包括：

  1. 监控网络性能：监控网络性能，包括网络带宽、网络延迟和网络流量等。
  2. 网络故障排除：解决网络故障，包括网络连接故障、网络设备故障和网络协议故障等。
  3. 网络配置管理：管理网络配置，包括IP地址、子网掩码和网关等。
  4. 网络安全管理：管理网络安全，包括防火墙配置、访问控制列表和入侵检测系统等。
  5. 网络设备管理：管理网络设备，包括路由器、交换机和防火墙等。
  6. 网络用户管理：管理网络用户，包括用户身份验证、用户权限和用户访问控制等。
  7. 网络备份和恢复：备份和恢复网络数据，确保网络数据的安全和可靠。

应用层

应用层是计算机网络体系结构中的最高层，它为用户提供各种网络服务。应用层协议包括HTTP、FTP、SMTP、DNS等。应用层协议负责处理用户请求，并将请求发送到网络层。

DNS

域名系统概述

域名系统（DNS）是互联网中的一个重要服务，它将域名解析为IP地址。DNS协议使用UDP协议进行通信，端口号为53。

域名结构

域名由多个标签组成，每个标签之间用点号（.）分隔。例如，www.example.com是一个域名，其中www是主机名，example是二级域名，com是顶级域名。

域名服务器

域名服务器是存储域名和IP地址映射关系的数据库。DNS协议使用域名服务器来解析域名。域名服务器分为以下几种类型：

  1. 根域名服务器：根域名服务器存储顶级域名的IP地址。
  2. 顶级域名服务器：顶级域名服务器存储二级域名的IP地址。
  3. 权限域名服务器：权限域名服务器存储特定域名的IP地址。
  4. 本地域名服务器：本地域名服务器是用户计算机上的DNS服务器，它负责将域名解析为IP地址。

DNS解析过程：

  1. 用户在浏览器中输入域名，如www.example.com。
  2. 浏览器向本地DNS服务器发送DNS查询请求，请求解析域名。
  3. 本地DNS服务器向根DNS服务器发送DNS查询请求，请求解析域名。
  4. 根DNS服务器返回顶级域名（如.com）的DNS服务器地址。
  5. 本地DNS服务器向顶级域名DNS服务器发送DNS查询请求，请求解析域名。
  6. 顶级域名DNS服务器返回二级域名（如example.com）的DNS服务器地址。
  7. 本地DNS服务器向二级域名DNS服务器发送DNS查询请求，请求解析域名。
  8. 二级域名DNS服务器返回域名对应的IP地址。
  9. 本地DNS服务器将IP地址返回给浏览器。
  10. 浏览器使用IP地址与服务器建立连接，并获取网页内容。

递归查询和迭代查询

DNS查询有两种类型：递归查询和迭代查询。

递归查询是指DNS服务器将查询结果返回给用户计算机。如果DNS服务器无法解析域名，它将向其他DNS服务器发送查询请求，直到找到IP地址为止。
迭代查询是指DNS服务器将查询结果返回给用户计算机。如果DNS服务器无法解析域名，它将向其他DNS服务器发送查询请求，直到找到IP地址为止。

DNS缓存

DNS缓存是指将域名和IP地址的映射关系存储在本地计算机上，以便快速解析域名。DNS缓存可以减少DNS查询的次数，提高网络性能。

DNS安全

DNS安全是保护DNS服务器和DNS数据免受攻击的重要方面。常见的DNS安全措施包括：

  1. 使用DNSSEC：DNSSEC是一种数字签名技术，用于验证DNS数据的完整性和真实性。
  2. 使用DNS加密：DNS加密是一种加密技术，用于保护DNS查询和响应的机密性。
  3. 使用DNS缓存：DNS缓存可以减少DNS查询的次数，提高网络性能。
  4. 使用DNS过滤：DNS过滤是一种技术，用于过滤DNS查询和响应，防止恶意域名解析。
  5. 使用DNS监控：DNS监控是一种技术，用于监控DNS服务器的状态和性能，及时发现和解决DNS问题。

文件传输协议

FTP（File Transfer Protocol，文件传输协议）是一种用于在计算机之间传输文件的协议。FTP使用TCP协议进行通信，端口号为20和21。

FTP工作原理：

  1. 客户端向FTP服务器发送连接请求，建立TCP连接。
  2. 客户端向FTP服务器发送用户名和密码，进行身份验证。
  3. 客户端向FTP服务器发送文件传输命令，如上传、下载或删除文件。
  4. FTP服务器接收到命令后，执行相应的操作，并将结果返回给客户端。
  5. 客户端接收到结果后，关闭连接。

FTP传输模式：

FTP协议有两种传输模式：主动模式和被动模式。
主动模式是指FTP服务器主动向客户端发送连接请求，建立TCP连接。主动模式需要客户端的防火墙允许外部连接。
被动模式是指FTP服务器被动地等待客户端的连接请求，建立TCP连接。被动模式需要客户端的防火墙允许外部连接。

FTP安全

FTP协议本身不提供加密功能，因此存在安全风险。为了保护数据传输的安全性，可以使用以下方法：

  1. 使用SFTP（Secure File Transfer Protocol，安全文件传输协议）：SFTP是一种基于SSH协议的文件传输协议，提供加密功能，可以保护数据传输的安全性。
  2. 使用FTPS（FTP Secure，安全FTP）：FTPS是一种基于SSL/TLS协议的FTP协议，提供加密功能，可以保护数据传输的安全性。
  3. 使用加密通道：可以使用VPN（Virtual Private Network，虚拟专用网络）等加密通道，保护数据传输的安全性。

简单文件传输协议

TFTP（Trivial File Transfer Protocol，简单文件传输协议）是一种用于在计算机之间传输文件的协议。FTP使用TCP协议进行通信，端口号为20和21。

TFTP优点：

  1. TFTP可用于UDP传输，无需建立TCP连接，传输速度快。
  2. TFTP代码量小，实现简单，适合嵌入式系统使用。

TFTP缺点：

  1. 不支持断点续传：TFTP协议不支持断点续传，如果传输过程中出现错误，需要重新传输整个文件。
  2. 不支持目录操作：TFTP协议不支持目录操作，只能传输文件。
  3. 不支持加密：TFTP协议本身不提供加密功能，因此存在安全风险。

TFTP的主要特点

  1. 每次传输的数据报文中有512字节的数据，但最后一次可不足512字节
  2. 数据报文按序编号，从1开始
  3. 支持ASCII码或二进制传送
  4. 可对文件进行读或写
  5. 使用简单的首部

远程终端协议 TELNET

TELNET（Teletype Network，电传网络）是一种用于远程登录的协议，允许用户通过计算机终端连接到远程计算机，并执行远程计算机上的命令。TELNET使用TCP协议进行通信，端口号为23。

TELNET的主要特点

  1. 支持多种操作系统和终端类型
  2. 支持多种字符集和编码
  3. 支持多种命令和操作
  4. 支持多种传输模式和协议
  5. 支持多种安全性和认证机制

万维网

万维网（World Wide Web，简称WWW）是一个基于超文本的分布式系统，允许用户通过浏览器访问和浏览互联网上的信息。万维网使用HTTP协议进行通信，端口号为80。

万维网的主要特点

  1. 超文本：万维网使用超文本技术，允许用户通过链接从一个页面跳转到另一个页面。
  2. 分布式：万维网是一个分布式系统，由多个服务器和客户端组成，用户可以通过浏览器访问和浏览互联网上的信息。
  3. 交互性：万维网支持用户与网页的交互，如填写表单、点击链接等。
  4. 多媒体：万维网支持多种媒体类型，如文本、图片、音频、视频等。
  5. 易用性：万维网使用户可以轻松地访问和浏览互联网上的信息，无需了解复杂的网络协议和技术。

统一资源定位符 URL

统一资源定位符（Uniform Resource Locator，简称URL）是万维网中用于标识资源的唯一标识符。URL由协议、域名、路径、查询参数和锚点组成，例如：http://www.example.com/path/to/resource?query=param#anchor

HTTP

HTTP（HyperText Transfer Protocol，超文本传输协议）是互联网中最常用的协议之一，用于在Web浏览器和Web服务器之间传输数据。HTTP协议使用TCP协议进行通信，端口号为80。

HTTP请求和响应：

  1. 客户端向服务器发送HTTP请求，请求服务器提供特定的资源，如网页、图片或视频等。
  2. 服务器接收到请求后，根据请求的内容返回相应的HTTP响应，如200 OK表示请求成功，404 Not Found表示请求的资源不存在等。
  3. 客户端接收到HTTP响应后，根据响应的内容进行相应的处理，如显示网页、下载文件或播放视频等。

HTTP请求方法：

  1. GET：请求指定的资源。
  2. POST：向指定的资源提交数据进行处理请求（例如提交表单或者上传文件）。
  3. PUT：向指定的资源位置上传其最新内容。
  4. DELETE：请求服务器删除指定的资源。
  5. HEAD：与GET请求相同，但服务器在响应中只返回头部信息，不返回实体的主体。
  6. OPTIONS：返回目标资源所支持的通信选项。
  7. PATCH：对资源进行部分修改。
  8. CONNECT：HTTP/1.1协议中预留给能够将连接改为管道方式的代理服务器。
  9. TRACE：回显服务器收到的请求，主要用于测试或诊断。

HTTP状态码：

1xx：信息性状态码，表示请求已被接收，继续处理。
2xx：成功状态码，表示请求已成功被服务器接收、理解、并接受。
3xx：重定向状态码，表示需要客户端采取进一步的操作才能完成请求。
4xx：客户端错误状态码，表示请求包含语法错误或无法被服务器理解。
5xx：服务器错误状态码，表示服务器在处理请求的过程中发生了错误。

电子邮件

电子邮件是一种通过互联网发送和接收消息的通信方式。电子邮件使用SMTP（Simple Mail Transfer Protocol，简单邮件传输协议）进行邮件的发送，使用POP3（Post Office Protocol version 3，邮局协议版本3）或IMAP（Internet Message Access Protocol，互联网消息访问协议）进行邮件的接收。

电子邮件的组成部分：

  1. 发件人：邮件的发件人地址，表示邮件的发送者。
  2. 收件人：邮件的收件人地址，表示邮件的接收者。
  3. 主题：邮件的主题，表示邮件的主要内容。
  4. 正文：邮件的正文，表示邮件的具体内容。
  5. 附件：邮件的附件，表示邮件中包含的额外文件。

电子邮件的工作流程：

  1. 用户通过邮件客户端编写邮件，并设置发件人和收件人地址。
  2. 邮件客户端使用SMTP协议将邮件发送到邮件服务器。
  3. 邮件服务器接收到邮件后，根据收件人地址将邮件发送到相应的邮件服务器。
  4. 收件人的邮件服务器接收到邮件后，根据用户的设置将邮件存储到用户的邮箱中。
  5. 用户通过邮件客户端使用POP3或IMAP协议从邮件服务器中下载邮件。

物理层

物理层的作用：连接不同的物理设备，传输比特流。该层为上层协议提供了一个传输数据的可靠的物理媒体。简单的说，物理层确保原始的数据可在各种物理媒体上传输。

物理层设备：

中继器【Repeater，也叫放大器】：同一局域网的再生信号；两端口的网段必须同一协议；5-4-3规程： 10BASE-5以太网中，最多串联4个中继器，5段中只能有3个连接主机；
集线器：同一局域网的再生、放大信号（多端口的中继器）；半双工，不能隔离冲突域也不能隔离广播域。
信道的基本概念：信道是往一个方向传输信息的媒体，一条通信电路包含一个发送信道和一个接受信道。

单工通信信道：只能一个方向通信，没有反方向反馈的信道；
半双工通信信道：双方都可以发送和接受信息，但不能同时发送也不能同时接收；
全双工通信信道：双方都可以同时发送和接收。
