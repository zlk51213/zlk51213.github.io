操作系统

操作系统概述

操作系统的概念

操作系统（Operating System，OS）是管理计算机硬件和软件资源的计算机程序，是计算机系统的核心。

操作系统的目标

  1. 方便性
  2. 有效性
  3. 可扩展性
  4. 开放性

操作系统的功能

  1. 进程管理：操作系统负责管理计算机系统的进程，包括进程的创建、调度、同步、通信和终止等。
  2. 内存管理：操作系统负责管理计算机系统的内存，包括内存的分配、回收、保护和共享等。
  3. 文件管理：操作系统负责管理计算机系统的文件，包括文件的创建、删除、读写、保护和共享等。
  4. 设备管理：操作系统负责管理计算机系统的设备，包括设备的分配、控制和通信等。
  5. 用户接口：操作系统负责提供用户接口，包括命令行接口和图形用户接口等，以便用户与计算机系统进行交互。

操作系统的历史

  1. 20世纪50年代：操作系统起源于20世纪50年代的大型计算机系统。当时，计算机系统主要用于科学计算和军事应用，操作系统的主要功能是管理计算机的硬件资源。
  2. 20世纪60年代：随着计算机技术的不断发展，操作系统开始应用于个人计算机。当时，操作系统的主要功能是管理计算机的硬件资源，并提供用户接口。
  3. 20世纪70年代：随着个人计算机的普及，操作系统开始出现各种不同的版本。当时，操作系统的主要功能是管理计算机的硬件资源，并提供用户接口。
  4. 20世纪80年代：随着计算机网络的兴起，操作系统开始出现网络操作系统。当时，操作系统的主要功能是管理计算机的硬件资源，并提供用户接口。
  5. 21世纪：随着云计算和大数据技术的兴起，操作系统开始出现分布式操作系统。当时，操作系统的主要功能是管理计算机的硬件资源，并提供用户接口。

操作系统的组成

  1. 内核：内核是操作系统的核心部分，负责管理计算机系统的硬件资源，包括CPU、内存、磁盘等。内核通常由一组系统调用组成，供应用程序使用。
  2. 系统调用：系统调用是操作系统提供的接口，供应用程序使用。应用程序通过系统调用请求操作系统提供的服务，如创建进程、读写文件等。
  3. 设备驱动程序：设备驱动程序是操作系统的一部分，负责管理计算机系统的硬件设备。设备驱动程序通常由硬件制造商提供。
  4. 文件系统：文件系统是操作系统的一部分，负责管理计算机系统的文件和目录。文件系统通常由一组文件操作组成，供应用程序使用。
  5. 用户界面：用户界面是操作系统的一部分，负责与用户进行交互。用户界面通常由一组图形界面或命令行界面组成。

操作系统的功能

  1. 进程管理：操作系统负责管理计算机系统的进程，包括进程的创建、调度、通信和同步。
  2. 内存管理：操作系统负责管理计算机系统的内存，包括内存的分配、回收和保护。
  3. 文件管理：操作系统负责管理计算机系统的文件，包括文件的创建、删除、读写和搜索。
  4. 设备管理：操作系统负责管理计算机系统的硬件设备，包括设备的初始化、配置和操作。
  5. 网络管理：操作系统负责管理计算机系统的网络，包括网络的配置、通信和安全管理。
  6. 用户接口：操作系统负责提供用户接口，包括命令行接口和图形用户接口等，以便用户与计算机系统进行交互。

操作系统的分类

  1. 实时操作系统：实时操作系统（Real-Time Operating System，RTOS）是一种能够及时响应外部事件并执行相应操作的操作系统。它通常用于需要实时响应的应用场景，如嵌入式系统和实时控制系统。
  2. 批处理操作系统：批处理操作系统（Batch Processing Operating System）是一种能够自动处理一批作业的操作系统。它通常用于大型计算机系统，如大型机。
  3. 分时操作系统：分时操作系统（Time-Sharing Operating System）是一种能够将计算机系统的资源分配给多个用户或进程的操作系统。它通常用于多用户环境，如个人计算机。
  4. 网络操作系统：网络操作系统（Network Operating System，NOS）是一种能够管理计算机网络的操作系统。它通常用于计算机网络，如局域网和广域网。
  5. 分布式操作系统：分布式操作系统（Distributed Operating System，DOS）是一种能够管理多个计算机系统的操作系统。它通常用于分布式计算环境，如云计算和大数据处理。

操作系统的发展

未配置操作系统的计算机系统

在计算机系统出现之初，没有配置操作系统的计算机系统只能执行单个程序，用户需要手动输入程序指令，效率低下。

  1. 人工操作方式：用户需要手动输入程序指令，然后等待计算机执行，效率低下。
  2. 脱机I/O方式：用户将程序和数据准备好后，将程序和数据输入计算机，然后计算机执行程序，执行完毕后，将结果输出到用户指定的设备上。

单道批处理系统

单道批处理系统是一种能够自动处理一批作业的操作系统。它通常用于大型计算机系统，如大型机。

  1. 作业：作业是指用户提交给计算机系统的一组程序和数据，计算机系统需要按照一定的顺序执行这些程序和数据。
  2. 批处理：批处理是指将一组作业按照一定的顺序输入计算机系统，计算机系统按照顺序执行这些作业。
  3. 优点：单道批处理系统可以自动处理一批作业，提高了计算机系统的资源利用率，提高了计算机系统的效率。
  4. 缺点：单道批处理系统只能同时处理一个作业，如果作业执行过程中出现错误，需要重新输入作业，效率较低。

多道批处理系统

多道批处理系统是一种能够同时处理多个作业的操作系统。它通常用于大型计算机系统，如大型机。
优缺点：

  1. 优点：多道批处理系统可以同时处理多个作业，提高了计算机系统的资源利用率，提高了计算机系统的效率。
  2. 缺点：多道批处理系统只能按照作业的顺序执行，如果作业执行过程中出现错误，需要重新输入作业，效率较低。

分时操作系统

分时操作系统是一种能够将计算机系统的资源分配给多个用户或进程的操作系统。它通常用于多用户环境，如个人计算机。

  1. 分时：分时是指将计算机系统的资源分配给多个用户或进程，每个用户或进程可以按照一定的时隙使用计算机系统的资源。
  2. 优点：分时操作系统可以同时处理多个用户或进程，提高了计算机系统的资源利用率，提高了计算机系统的效率。
  3. 缺点：分时操作系统需要为每个用户或进程分配一定的资源，如果用户或进程需要更多的资源，需要等待其他用户或进程释放资源，效率较低。

实时操作系统

实时操作系统是一种能够及时响应外部事件并执行相应操作的操作系统。它通常用于实时控制环境，如工业控制、航空航天和医疗设备。

  1. 实时控制：实时控制是指计算机系统需要及时响应外部事件并执行相应操作，以满足实时控制的要求。
  2. 优点：实时操作系统可以及时响应外部事件并执行相应操作，满足实时控制的要求。
  3. 缺点：实时操作系统需要及时响应外部事件并执行相应操作，如果外部事件频繁发生，计算机系统需要频繁切换任务，效率较低。

微内核操作系统

微内核操作系统是一种将操作系统的主要功能模块化，并将一些功能模块移到用户空间的操作系统。它通常用于高性能计算和嵌入式系统。

  1. 微内核：微内核是指操作系统的主要功能模块化，并将一些功能模块移到用户空间。
  2. 优点：微内核操作系统可以提高系统的可扩展性和可维护性，提高系统的安全性。
  3. 缺点：微内核操作系统需要更多的用户空间资源，如果用户空间资源不足，系统性能可能会受到影响。

嵌入式操作系统

嵌入式操作系统是一种用于嵌入式系统的操作系统。它通常用于嵌入式设备，如智能手机、平板电脑、汽车电子设备等。

  1. 嵌入式系统：嵌入式系统是指将计算机系统嵌入到其他设备中，以实现特定功能。
  2. 优点：嵌入式操作系统可以提高设备的性能和可靠性，降低设备的成本。
  3. 缺点：嵌入式操作系统需要根据设备的需求进行定制，开发周期较长。

网络操作系统

网络操作系统是一种用于网络环境的操作系统。它通常用于计算机网络，如局域网、广域网等。

  1. 网络环境：网络环境是指计算机系统通过网络进行通信和资源共享的环境。
  2. 优点：网络操作系统可以提高计算机系统的通信效率和资源共享能力，提高计算机系统的可用性和可靠性。
  3. 缺点：网络操作系统需要更多的网络资源，如果网络资源不足，系统性能可能会受到影响。

分布式操作系统

分布式操作系统是一种将计算机系统通过网络连接起来，形成一个整体系统的操作系统。它通常用于分布式计算和云计算环境。

  1. 分布式计算：分布式计算是指将计算任务分配给多个计算机系统，以实现更高的计算性能。
  2. 优点：分布式操作系统可以提高计算机系统的计算性能和可靠性，提高计算机系统的可用性和可扩展性。
  3. 缺点：分布式操作系统需要更多的网络资源，如果网络资源不足，系统性能可能会受到影响。

操作系统的基本特性

并发

并发：指两个或多个事件可以在同一个时间间隔发生，用户看起来是每个程序都在运行，实际上是每个程序都交替执行。
并行：指两个或多个事件可以在同一个时刻发生，多核CPU可以实现并行，一个cpu同一时刻只有一个程序在运行；

共享

操作系统允许多个进程或线程共享计算机系统的资源，如CPU、内存、输入输出设备等。

  1. 互斥共享：多个进程或线程需要互斥地访问共享资源，以避免资源冲突。
  2. 同时共享：多个进程或线程可以同时访问共享资源，以提高系统性能。

虚拟

操作系统通过虚拟化技术，将物理资源映射为虚拟资源，为进程或线程提供独立的运行环境。
运用的技术：

  1. 时分复用技术
  2. 空间复用技术

异步

操作系统支持异步操作，即进程或线程可以在等待某个操作完成时继续执行其他操作。

操作系统的中断处理

中断机制的作用：为了在多道批处理系统中让用户进行交互

中断产生：

发生中断时，CPU立马切换到管态，开展管理工作；（管态又叫特权态，系统态或核心态，是操作系统管理的程序执行时，机器所处的状态。）
发生中断后，当前运行的进程回暂停运行，由操作系统内核对中断进行处理；
对于不同的中断信号，会进行不同的处理

中断处理过程：

保护现场：将当前进程的CPU环境保存起来，以便中断处理结束后恢复；
中断处理：根据中断信号，执行相应的中断处理程序；
恢复现场：将保存的CPU环境恢复，继续执行被中断的进程；
中断返回：返回到被中断的进程，继续执行。

中断的分类

内中断：由CPU检测到的异常事件，如除零错误、非法指令等。
外中断：由外部设备发出的中断信号，如时钟中断、I/O中断等。
外中断的处理过程
1.每执行完一个指令后，CPU都需要检查当前是否有外部中断 信号；
2。如果检查到外部中断信号，则需要保护被中断进程的CPU环境（如程序状态字PSW，程序计数器PC、各种通用寄存器）把他们存储在PCB（进程控制块中）；
3.根据中断信号类型转入相应的中断处理程序；
4.恢复原进程的CPU环境并退出中断，返回原进程继续执行。

操作系统的主要功能

处理机管理功能

  1. 进程控制：创建、撤销进程，以及进程状态的转换；
  2. 进程同步：解决进程间的竞争和协作问题；
  3. 进程通信：实现进程间的数据交换和消息传递；
  4. 调度：决定进程的执行顺序，提高系统资源利用率；

存储器管理功能

  1. 内存分配和回收：为进程分配内存空间，实现内存保护；
  2. 内存保护：防止进程间非法访问内存；
  3. 内存扩充：通过虚拟内存技术，实现内存的扩充；
  4. 地址映射：将逻辑地址映射为物理地址，实现内存访问；

设备管理功能

  1. 缓冲管理：通过缓冲区，提高I/O效率；
  2. 设备分配：为进程分配所需的I/O设备；
  3. 设备处理：处理I/O请求，实现设备与进程的交互；

文件管理功能

  1. 文件存储空间管理：分配和回收文件存储空间；
  2. 文件读写操作：实现文件的读写操作；
  3. 文件目录管理：实现文件的查找、创建、删除等操作；
  4. 文件保护：实现文件的安全性和完整性保护。

接口管理功能

  1. 用户接口：提供用户与操作系统交互的界面，如命令行、图形界面等；
  2. 程序接口：提供应用程序与操作系统交互的接口，如系统调用、库函数等；

操作系统的结构

简单结构

1. 内核：负责进程管理、内存管理、设备管理等核心功能；
2. 外壳：提供用户接口，实现用户与内核的交互；

模块化结构

1. 内核：负责进程管理、内存管理、设备管理等核心功能；
2. 模块：实现特定的功能，如文件管理、网络通信等；
3. 外壳：提供用户接口，实现用户与内核、模块的交互；

优缺点：
优点：模块化设计，易于维护和扩展；
缺点：模块间通信复杂，可能导致性能下降。

分层式结构

1. 用户层：提供用户接口，实现用户与操作系统的交互；
2. 系统调用层：实现系统调用，提供应用程序与内核的接口；
3. 内核层：负责进程管理、内存管理、设备管理等核心功能；
4. 设备驱动层：实现设备驱动程序，实现设备与内核的交互；

优缺点：
优点：层次清晰，易于理解和维护；
缺点：层次过多，可能导致性能下降。

微内核结构

1. 微内核：只提供最基本的功能，如进程管理、内存管理等；
2. 客户进程：实现特定的功能，如文件管理、网络通信等；

优缺点：
优点：模块化设计，易于维护和扩展；
缺点：微内核功能有限，可能需要更多的客户进程来实现复杂功能。

外核结构

1. 内核：负责进程管理、内存管理、设备管理等核心功能；
2. 外核：提供用户接口，实现用户与内核的交互；

优缺点：
优点：模块化设计，易于维护和扩展；
缺点：外核功能有限，可能需要更多的内核来实现复杂功能。

进程管理

进程概念

  1. 进程是程序在计算机上的一次执行过程，是系统进行资源分配和调度的基本单位；
  2. 进程具有独立的地址空间，可以并发执行，实现多任务处理；
  3. 进程之间可以通过通信、同步、互斥等机制实现协作和协调。

进程（Process）与线程（Thread）：

线程：操作系统进行运行调度的最小单位。
进程：系统进行资源分配和调度的基本单位。

区别与联系：
一个进程可以有一个或多个线程；
线程包含在进程之中，是进程中实际运行工作的单位；
进程的线程共享进程资源；
一个进程可以并发多个线程，每个线程执行不同的任

进程特征：

    动态性：进程是动态产生、消亡的；
    并发性：进程可以并发执行，实现多任务处理；
    独立性：进程具有独立的地址空间，可以独立运行；
    异步性：进程的执行是异步的，进程之间需要通过通信、同步、互斥等机制实现协作和协调。

进程状态

  1. 就绪状态：进程已经获取了除CPU外的所有资源，等待CPU调度；
  2. 运行状态：进程正在使用CPU，执行程序指令；
  3. 阻塞状态：进程等待某个事件（如I/O操作、等待资源）的发生，无法继续执行；

进程调度

  1. 先来先服务调度算法：按照进程到达的顺序进行调度；
  2. 最短作业优先调度算法：按照进程执行所需时间的长短进行调度；
  3. 时间片轮转调度算法：将CPU时间划分为固定长度的时间片，按照进程到达的顺序进行调度；
  4. 多级反馈队列调度算法：将进程分为多个队列，按照队列的优先级进行调度。

进程通信

  1. 管道通信：用于具有亲缘关系的进程之间的通信；
  2. 套接字通信：用于网络通信；
  3. 消息队列通信：用于进程间的数据交换；
  4. 共享内存通信：用于进程间的数据共享；
  5. 信号量通信：用于进程间的同步和互斥。

处理机调度与死锁

进程调度

定义：指计算机通过决策决定哪个就绪进程可以获得CPU使用权。
什么时候需要进程调度？

主动放弃：进程正常终止；运行过程中发生异常而终止；主动阻塞（如等待I/O）；
被动放弃：分给进程的时间片用完；有更高优先级的进程进入就绪队列；有更紧急的事情需要处理（如I/O中断）；

进程调度方式：

非抢占式调度：只能由当前运行的进程主动放弃CPU；
    处理器一旦分配给某个进程，就让该进程一直使用下去；
    调度程序不以任何原因抢占正在被使用的处理器；
    调度程序不以任何原因抢占正在被使用的处理器；
抢占式调度：可由操作系统剥夺当前进程的CPU使用权。
    允许调度程序以一定的策略暂停当前运行的进程；
    保存好旧进程的上下文信息，分配处理器给新进程；

进程调度的三大机制：

就绪队列的排队机制：为了提高进程调度的效率，将就绪进程按照一定的方式排成队列，以便调度程序可以最快找到就绪进程。

进程调度算法

  1. 先来先服务（FCFS）：按照进程到达的顺序进行调度；
  2. 最短作业优先（SJF）：按照进程执行所需时间的长短进行调度；
  3. 最短剩余时间优先（SRTF）：按照进程剩余执行时间的长短进行调度；
  4. 时间片轮转（RR）：将CPU时间划分为固定长度的时间片，按照进程到达的顺序进行调度；
  5. 多级反馈队列（MFQ）：将进程分为多个队列，按照队列的优先级进行调度。

进程死锁

进程死锁是指多个进程在等待对方释放资源时，导致无法继续执行的情况；
导致进程死锁的原因包括资源竞争、循环等待、不释放已获得的资源等；
解决进程死锁的方法包括预防死锁、避免死锁、检测死锁和解除死锁等。
死锁的四个必要条件：
互斥条件：必须互斥使用资源才会产生死锁；
请求保持条件：进程至少保持一个资源，又提出新的资源请求，新资源被占用，请求被阻塞，被阻塞的进程不释放自己保持的资源；
不可剥夺条件：进程获得的资源在未完成使用前不能被剥夺（包括OS），只能由进程自身释放；
环路等待条件：发生死锁时，必然存在进程-资源环形链,环路等待不一定造成死锁，但是死锁一定有循环等待。

死锁的处理决策

一.预防死锁的方法：破坏四个必要条件的中一个或多个。

破坏互斥条件：将临界资源改造成共享资源（Spooling池化技术）；（可行性不高，很多时候无法破坏互斥条件）
破坏请求保持条件：系统规定进程运行之前，一次性申请所有需要的资源；（资源利用率低，可能导致别的线程饥饿）
破坏不可剥夺条件：当一个进程请求新的资源得不到满足时，必须释放占有的资源；（实现复杂，剥夺资源可能导致部分工作失效，反复申请和释放造成额外的系统开销）
破坏环路等待条件：可用资源线性排序，申请必须按照需要递增申请；（进程实际使用资源顺序和编号顺序不同，会导致资源浪费）
二.银行家算法：检查当前资源剩余是否可以满足某个进程的最大需求；如果可以，就把该进程加入安全序列，等待进程允许完成，回收所有资源；重复1，2，直到当前没有线程等待资源；

三.死锁的检测和解除：死锁检测算法，资源剥夺法，撤销进程法（终止进程法），进程回退法；

进程同步

互斥锁：用于保护临界区，防止多个进程同时访问共享资源；
信号量：用于进程间的同步和互斥，可以控制多个进程对共享资源的访问；
事件：用于进程间的同步，可以等待某个事件的发生；
条件变量：用于进程间的同步，可以等待某个条件变量的满足。

进程同步与互斥

  1. 进程同步：进程间的协调和协作，确保进程按照一定的顺序执行；
  2. 进程互斥：进程间的竞争和限制，防止多个进程同时访问共享资源；
  3. 临界区：进程访问共享资源的一段代码；
  4. 信号量：用于进程间的同步和互斥，可以控制多个进程对共享资源的访问；
  5. 管程：用于进程间的同步和互斥，可以保护临界区，防止多个进程同时访问共享资源。

信道

单工通信信道：只能一个方向通信，没有反方向反馈的信道；
半双工通信信道：双方都可以发送和接受信息，但不能同时发送也不能同时接收；
全双工通信信道：双方都可以同时发送和接收。

常用编码方式

  1. 不归零编码：正电平代表1，负电平代表0。
  2. 归零编码：正脉冲代表1，负脉冲代表0。
  3. 曼彻斯特编码：位周期中心的向上跳代表0，位周期中心的向下跳代表1。
  4. 差分曼彻斯特编码：每一位的中间都有一条电平跳变，位开始边界有跳变代表0，位开始边界无跳变代表1。

基本带通调制方法

  1. 调幅：改变载波的幅度。
  2. 调频：改变载波的频率。
  3. 调相：改变载波的相位。

信道的极限容量

信道的极限容量是指信道能够传输的最大数据速率，即信道所能传输的最高数据率。

信道能通过的控制范围

奈氏准则指出，在理想低通信道的条件下，为了避免码间串扰，极限数据传输速率（即信道的最大数据传输速率）为：
C = 2H log2V bps
其中：
C：信道的极限容量，单位为比特/秒（bps）
H：信道带宽，单位为赫兹（Hz）
V：每个信号所用的码元个数
奈氏准则适用于理想低通信道，即信道的带宽足够宽，且信道的信噪比足够高。在实际应用中，为了满足奈氏准则，需要采取适当的编码方式，如差分曼彻斯特编码等。

信道的极限容量可以用以下公式计算：
C = B log2(1 + S/N)

其中：
C：信道的极限容量，单位为比特/秒（bps）
B：信道的带宽，单位为赫兹（Hz）
S：信号功率
N：噪声功率

信号量机制

信号量机制是一种用于进程同步和互斥的机制，它通过一个整型变量（称为信号量）来控制对共享资源的访问。
信号量机制的基本思想是：当一个进程需要访问共享资源时，它首先检查信号量的值。如果信号量的值大于0，则表示有可用的资源，进程可以继续执行。如果信号量的值等于0，则表示没有可用的资源，进程需要等待，直到有其他进程释放资源为止。
当进程访问完共享资源后，它会增加信号量的值，以表示释放了资源。这样，其他等待的进程就可以继续执行了。
信号量机制可以确保多个进程对共享资源的访问是互斥的，即同一时刻只有一个进程可以访问共享资源。这样可以避免多个进程同时访问共享资源时出现的数据竞争和死锁等问题。

信号量机制的基本操作包括P操作和V操作：

P操作：将信号量的值减1，如果信号量的值小于0，则进程需要等待，直到有其他进程释放资源为止。
V操作：将信号量的值加1，如果有等待的进程，则唤醒其中一个进程。

信号量机制的应用

信号量机制可以用于实现进程同步和互斥，例如：

  1. 进程同步：当一个进程需要等待另一个进程完成某个操作后才能继续执行时，可以使用信号量机制来实现进程同步。
  2. 进程互斥：当一个进程需要访问共享资源时，可以使用信号量机制来实现进程互斥，即同一时刻只有一个进程可以访问共享资源。
  3. 资源池：当一个进程需要访问有限数量的资源时，可以使用信号量机制来实现资源池，即限制对资源的访问数量，以避免资源耗尽。

信号量机制的特点

  1. 信号量机制是一种用于进程同步和互斥的机制，它通过一个整型变量（称为信号量）来控制对共享资源的访问。
  2. 信号量机制的基本操作包括P操作和V操作，P操作将信号量的值减1，V操作将信号量的值加1。
  3. 信号量机制可以确保多个进程对共享资源的访问是互斥的，即同一时刻只有一个进程可以访问共享资源。
  4. 信号量机制可以用于实现进程同步、进程互斥和资源池等功能。

信道复用技术

信道复用技术是一种将多个信道合并为一个信道的技术，以提高通信效率和资源利用率。常见的信道复用技术包括频分复用（FDM）、时分复用（TDM）、波分复用（WDM）和码分复用（CDM）等。
频分复用（FDM）：将一个信道分成多个频率不同的子信道，每个子信道传输不同的信号。频分复用技术可以同时传输多个信号，但每个信号只能占用一个子信道。
时分复用（TDM）：将一个信道分成多个时间片，每个时间片传输不同的信号。时分复用技术可以同时传输多个信号，但每个信号只能在一个时间片内传输。
波分复用（WDM）：将一个信道分成多个波长不同的子信道，每个子信道传输不同的信号。波分复用技术可以同时传输多个信号，但每个信号只能占用一个子信道。
码分复用（CDM）：将一个信道分成多个码片，每个码片传输不同的信号。码分复用技术可以同时传输多个信号，但每个信号只能占用一个码片。

信道复用技术的特点

  1. 信道复用技术是一种将多个信道合并为一个信道的技术，以提高通信效率和资源利用率。
  2. 频分复用（FDM）、时分复用（TDM）、波分复用（WDM）和码分复用（CDM）是常见的信道复用技术。
  3. 频分复用技术将一个信道分成多个频率不同的子信道，每个子信道传输不同的信号。
  4. 时分复用技术将一个信道分成多个时间片，每个时间片传输不同的信号。
  5. 波分复用技术将一个信道分成多个波长不同的子信道，每个子信道传输不同的信号。
  6. 码分复用技术将一个信道分成多个码片，每个码片传输不同的信号。

死锁

死锁是指两个或多个进程在执行过程中，因争夺资源而造成的一种互相等待的现象，导致这些进程都无法继续执行。死锁的产生通常是由于以下原因：

  1. 资源竞争：当多个进程需要同时访问同一资源时，如果没有适当的同步机制，可能会导致死锁。
  2. 资源不可抢占：当进程已经获得一个资源，但无法继续执行时，该资源将无法被其他进程抢占，从而导致死锁。
  3. 循环等待：当多个进程互相等待对方释放资源时，如果没有适当的同步机制，可能会导致死锁。
